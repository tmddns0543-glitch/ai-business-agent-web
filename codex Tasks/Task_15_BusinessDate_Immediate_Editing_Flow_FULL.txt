AI Business Agent 2세대 웹서비스의 다음 UX 수정 Task를 진행해줘.

이번 Task에서는 현재 구현된 `미리보기 날짜 → 이 날짜로 마감하기 → 하위 입력 시작` 구조를 제거하고, 다음 흐름으로 단순화한다.

날짜 선택
→ 해당 날짜를 즉시 활성 영업일로 저장
→ 해당 날짜의 매출·정산·기존 입력값 즉시 조회
→ 신규 입력 또는 기존 값 수정
→ 각 상세 화면에서 저장
→ 마감 화면 가장 하단의 최종 마감 버튼

이번 Task에서는 날짜별 마감 상태 저장소를 새로 구현하지 않는다.

즉 이번 작업의 목적은 날짜 선택 UX를 다시 단순화하고, 상단의 `이 날짜로 마감하기` 버튼을 제거하는 것이다.

최종 마감 완료 상태는 다음 Task에서 날짜별 상태 저장소와 함께 구현한다.

# 1. 확정 UX 원칙

날짜 선택은 저장이나 최종 확정 행동이 아니다.

날짜 선택의 의미:
사용자가 조회·입력·수정할 영업일을 선택

따라서 날짜 input이 변경되면 즉시 다음이 실행되어야 한다.

날짜 검증
→ setSelectedBusinessDate(선택 날짜)
→ business-day-context 저장
→ 현재 화면 state 변경
→ 해당 날짜 정산 결과 조회
→ 해당 날짜의 매출 카드 표시

사용자는 별도의 날짜 확정 버튼을 누르지 않는다.

# 2. 제거할 구조

다음 구조를 제거한다.

draftBusinessDate

미리보기 날짜 안내
아직 마감 영업일로 저장되지 않았다는 안내
미확정 날짜에서 카드 이동 차단
날짜 확정 성공 메시지
날짜 확정 실패 메시지

가능하면 날짜 state는 하나만 사용한다.

selectedBusinessDate

하위 입력 화면에서 사용하는 날짜

# 3. /closing 날짜 변경 동작

수정 파일:
frontend/src/app/closing/page.tsx

날짜 input 변경 시 다음을 실행한다.

입력 날짜 검증
→ setSelectedBusinessDate(date)
→ 저장 성공 시 state 갱신
→ 해당 날짜의 정산 summary 조회

저장 실패 시 기존 날짜를 유지하고 오류 메시지를 보여준다.

권장 오류 문구:
영업일을 변경하지 못했습니다. 다시 시도해주세요.

정상 변경 시 별도 성공 메시지는 필수가 아니다.

날짜 아래 안내는 단순하게 유지한다.

예:
2026년 7월 16일 마감 데이터를 보고 있습니다.

또는:
선택한 영업일의 매출·비용·마감 데이터를 확인합니다.

미리보기라는 표현은 사용하지 않는다.

# 4. 날짜 선택 후 즉시 조회·입력·수정 가능

날짜가 변경되는 즉시 해당 날짜의 카드로 이동할 수 있어야 한다.

다음 동작을 복구한다.

매출 카드 클릭 가능
비용 카드 클릭 가능
배달대행사 카드 클릭 가능
다음 단계 버튼 동작 가능

Task 14에서 추가한 미확정 날짜 이동 차단 로직을 제거한다.

현재 비용·배달대행사 화면이 아직 구현되지 않았다면 기존 링크 또는 비활성 상태를 그대로 유지하되, 날짜 미확정 때문에 추가로 막는 로직만 제거한다.

# 5. 신규 입력·조회·수정 흐름 검증

현재 날짜별 매출 저장소를 기준으로 다음 세 흐름이 모두 가능해야 한다.

## 신규 입력

마감
→ 날짜 선택
→ 매출
→ 플랫폼 선택
→ 빈 입력값
→ 값 입력
→ 저장

## 조회

마감
→ 날짜 선택
→ 해당 날짜의 총매출·예상 공제액·예상 정산금액 즉시 표시
→ 매출
→ 플랫폼별 기존 값 확인

## 수정

마감
→ 날짜 선택
→ 매출
→ 플랫폼 선택
→ 기존 입력값 자동 호출
→ 값 변경
→ 저장
→ 해당 날짜 정산 결과 갱신

별도의 수정 모드 버튼은 만들지 않는다.

기존 데이터가 있으면 자동으로 폼에 불러오고, 다시 저장하면 같은 날짜·플랫폼 데이터를 갱신하는 현재 구조를 유지한다.

# 6. 최종 마감 버튼 위치

현재 상단에 있던 `이 날짜로 마감하기` 버튼은 제거한다.

마감 화면 가장 하단에는 최종 마감 행동을 위한 버튼 영역을 둔다.

다만 날짜별 마감 상태 저장소가 아직 없으므로 이번 Task에서는 실제 closing completed 상태 구조를 새로 만들지 않는다.

현재 기존 마감 완료 버튼이 이미 하단에 있다면 현재 위치와 기능을 유지하되 버튼 문구와 위치를 점검한다.

권장 문구:
마감 완료하기

이미 기존 문구가 자연스럽다면 변경하지 않아도 된다.

중요:
상단 날짜 확정 버튼은 제거
하단 최종 마감 버튼은 유지

이번 Task에서 하단 버튼의 날짜별 저장 기능은 구현하지 않는다.
