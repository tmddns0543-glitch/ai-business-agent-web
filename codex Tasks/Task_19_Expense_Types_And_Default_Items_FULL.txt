AI Business Agent 2세대 웹서비스의 다음 구현 Task를 진행해줘.

이번 Task에서는 비용 기능의 첫 번째 기반 작업만 진행한다.

구현 범위:

1. 비용 관련 TypeScript 타입 정의
2. 기본 비용 카테고리 정의
3. 기본 비용 항목 상수 정의
4. 사용자 정의 비용 항목 타입 정의
5. 일일 비용 메모 타입 정의
6. 비용 항목별 재무 영향 규칙 정의

아직 LocalStorage 저장소, 비용 화면, 입력 폼, 합계 계산 화면은 만들지 않는다.

이번 Task의 목적은 앞으로 구현할 날짜별 비용 저장소와 비용 입력 UI가 흔들리지 않도록 비용 데이터 구조와 공식 기본 항목을 먼저 확정하는 것이다.

# 1. 생성할 타입 파일

다음 파일을 생성한다.

frontend/src/types/expense.ts

다음 타입을 정의한다.

## ExpenseGroup

공식 비용 그룹:

- material-purchase
- platform-cost
- fixed-cost
- operating-cost
- labor-cost
- tax-payment

권장 타입:

export type ExpenseGroup =
  | "material-purchase"
  | "platform-cost"
  | "fixed-cost"
  | "operating-cost"
  | "labor-cost"
  | "tax-payment";

## ExpenseTransactionType

초기 MVP에서는 일반 지출만 우선 사용하지만 향후 환급·취소 확장을 위해 다음 구조를 허용한다.

export type ExpenseTransactionType =
  | "expense"
  | "refund"
  | "cancellation";

다만 이번 Task에서는 실제 계산 로직을 만들지 않는다.

## ExpenseTransaction

필수 필드:

- id
- businessDate
- group
- itemId
- itemName
- amount
- transactionType
- createdAt
- updatedAt

선택 필드:

- vendorName
- platformId
- memo
- estimatedInputVat
- evidenceId
- evidenceType
- ocrStatus
- inventoryApplied

권장 예시:

export interface ExpenseTransaction {
  id: string;
  businessDate: BusinessDate;

  group: ExpenseGroup;
  itemId: ExpenseItemId;
  itemName: string;

  amount: number;
  transactionType: ExpenseTransactionType;

  vendorName?: string;
  platformId?: ExpensePlatformId;
  memo?: string;

  estimatedInputVat?: number;

  evidenceId?: string;
  evidenceType?: ExpenseEvidenceType;
  ocrStatus?: ExpenseOcrStatus;
  inventoryApplied?: boolean;

  createdAt: string;
  updatedAt: string;
}

`BusinessDate`는 기존 타입을 import한다.

## ExpensePlatformId

플랫폼 비용에서 사용하는 공식 플랫폼:

- baemin
- coupang-eats
- yogiyo
- ddangyo

가능하면 기존 `SettlementPlatformId`를 재사용하거나 그중 일반결제를 제외한 타입을 안전하게 파생한다.

중복 문자열 유니온을 무분별하게 만들지 않는다.

일반결제는 플랫폼 광고비·쿠폰비 입력 대상이 아니므로 포함하지 않는다.

## ExpenseEvidenceType

향후 증빙 확장:

- receipt
- invoice
- statement
- other

## ExpenseOcrStatus

향후 OCR 확장:

- none
- pending
- completed
- failed

현재 UI에는 노출하지 않는다.

# 4. 비용 항목 타입

다음 타입을 정의한다.

## ExpenseItemId

공식 기본 비용 항목 ID 문자열 유니온 또는 상수 기반 타입.

ID는 사람이 보는 label과 분리한다.

권장 ID 형식:

material-purchase.purchase
platform-cost.advertising
platform-cost.coupon
fixed-cost.rent
fixed-cost.electricity
operating-cost.supplies
labor-cost.payroll
tax-payment.vat

현재 프로젝트의 기존 ID naming convention이 있다면 그 스타일을 따른다.

## ExpenseItemDefinition

필드:

- id
- group
- label
- enabled
- system
- supportsPlatformBreakdown
- supportsVendor
- supportsMemo
- estimatesInputVat
- affectsCashFlow
- affectsOperatingProfit
- affectsTaxEstimate

권장:

export interface ExpenseItemDefinition {
  readonly id: ExpenseItemId;
  readonly group: ExpenseGroup;
  readonly label: string;

  readonly enabled: boolean;
  readonly system: boolean;

  readonly supportsPlatformBreakdown: boolean;
  readonly supportsVendor: boolean;
  readonly supportsMemo: boolean;
  readonly estimatesInputVat: boolean;

  readonly affectsCashFlow: boolean;
  readonly affectsOperatingProfit: boolean;
  readonly affectsTaxEstimate: boolean;
}

필요하면 `createsAsset`, `affectsInventory` 필드를 추가할 수 있다.

다만 이번 비용 화면에는 초기 투자·자산을 넣지 않으므로 기본 비용 항목에서는 `createsAsset`가 모두 false가 될 수 있다.

원재료 매입은 향후 재고 연동을 위해 `affectsInventory: true`를 허용한다.

# 8. 기본 항목 helper 함수

다음 역할의 순수 함수를 제공한다.

가능한 파일:

frontend/src/data/expense-default-items.ts

또는:

frontend/src/lib/expense/expense-item-utils.ts

필요 함수:

getDefaultExpenseItemsByGroup(group)

- 공식 순서 유지
- 해당 그룹 항목만 반환
- 새 배열 반환

getDefaultExpenseItem(itemId)

- ID로 기본 항목 조회
- 없으면 undefined

isCustomizableExpenseGroup(group)

- 사용자 정의 항목 허용 여부 반환
- material-purchase는 false
- 나머지는 true

getExpenseGroupDefinition(group)

- 그룹 정보 반환

이번 Task에서는 합계 계산, VAT 계산, LocalStorage 접근은 하지 않는다.

# 9. ID와 label 분리

다음 원칙을 반드시 지킨다.

- 저장과 참조는 itemId 사용
- 화면 표시는 label 사용
- itemName은 거래 저장 당시 label 보존용
- 사용자 정의 항목이 비활성화되어도 과거 거래 itemName 유지 가능

기본 항목 label 변경이 과거 거래 표시를 깨뜨리지 않도록 설계한다.

# 10. readonly와 불변성

기본 그룹·항목 상수는 외부에서 수정되지 않도록 readonly 구조를 사용한다.

helper 함수는 원본 배열과 객체를 직접 반환해 외부에서 mutate될 위험이 있다면 복사본을 반환한다.

입력 객체를 변경하지 않는다.

# 11. 테스트 또는 런타임 검증

새 테스트 패키지를 설치하지 않는다.

현재 테스트 환경이 없다면 임시 런타임 검증을 수행하고 검증 파일은 삭제한다.

필수 검증:

1. 공식 비용 그룹 6개 존재
2. 공식 순서 정확
3. 모든 기본 항목 ID 중복 없음
4. 모든 항목의 group이 공식 그룹에 속함
5. 플랫폼 비용은 광고비·쿠폰비만 존재
6. 공과금·고정비 11개 항목 존재
7. 일반 운영비에 기타잡비 없음
8. 인건비 3개 항목 존재
9. 세금 납부 5개 항목 존재
10. 세금 납부의 affectsOperatingProfit이 false
11. 원재료 매입 supportsVendor와 supportsMemo가 true
12. 원재료 매입 affectsInventory가 true
13. 플랫폼 비용 supportsPlatformBreakdown이 true
14. material-purchase는 사용자 정의 항목 그룹에서 제외
15. helper 함수가 공식 순서를 유지
16. helper 반환값 변경이 원본 상수에 영향을 주지 않음
17. TypeScript 컴파일 통과
