AI Business Agent 2세대 웹서비스의 다음 UX 수정 Task를 진행해줘.

이번 Task에서는 `/closing` 화면의 날짜 중복 표시를 정리하고, 화면 최하단에 선택 영업일이 명확히 표시되는 최종 마감 버튼 영역을 추가한다.

중요:
- 이번 Task에서는 날짜별 마감 상태 저장소를 구현하지 않는다.
- 기존 전역 완료 상태와 진행률 로직도 변경하지 않는다.
- 이번 작업은 화면 구조와 버튼 UX를 정리하는 단계다.
- 날짜별 최종 마감 저장은 다음 Task에서 구현한다.

# 1. 날짜 중복 표시 정리

현재 같은 날짜가 다음처럼 여러 번 노출되고 있다.

1. 영업일 date input
2. date input 아래 `2026년 7월 15일 마감 데이터를 보고 있습니다.`
3. `오늘 마감` 제목 위 `2026년 7월 15일 영업일`

이번 Task에서는 중복을 줄인다.

최종 권장 구조:

영업일
[ 2026-07-15 ]

오늘 마감
원하는 항목부터 확인해도 괜찮아요.

즉 다음 문구는 제거한다.

- date input 아래 `2026년 7월 15일 마감 데이터를 보고 있습니다.`
- 제목 위 `2026년 7월 15일 영업일`

date input 자체가 현재 영업일을 충분히 보여주므로 같은 날짜를 텍스트로 반복하지 않는다.

# 2. 제목 처리

기본적으로 기존 `오늘 마감`을 유지한다.

다만 과거 날짜에서도 `오늘`이라는 표현이 명백히 어색하면 다음 중 하나로 변경할 수 있다.

- 마감
- 영업일 마감

가장 단순하고 현재 디자인을 해치지 않는 쪽을 선택한다.

완료 보고에서 어떤 문구를 사용했는지 설명한다.

# 3. 최하단 최종 마감 영역 추가

`/closing` 화면의 모든 카드와 단순 잔액 영역 아래에 최종 마감 영역을 추가한다.

위치:

매출 카드
→ 비용 카드
→ 배달대행사 카드
→ 현재 단순 잔액
→ 기존 단계 이동 버튼
→ 최종 마감 안내
→ 선택 날짜가 들어간 최종 마감 버튼

하단 내비게이션보다 위에 위치해야 한다.

충분한 하단 여백을 둬서 버튼이 하단 내비게이션에 가리지 않게 한다.

# 4. 최종 마감 버튼 문구

버튼에는 현재 선택 영업일의 월·일을 표시한다.

예:

7월 15일 마감 완료하기

날짜는 `selectedBusinessDate`를 기반으로 동적으로 생성한다.

연도는 생략한다.

사용 금지:

- 오늘 매출 마감하기
- 오늘 저장하기
- 최종 저장
- 날짜 저장
- 이 날짜로 마감하기

공식 문구:

7월 15일 마감 완료하기

# 5. 버튼 활성 조건

현재 마감 단계는 다음 세 가지다.

- 매출 확인
- 비용 확인
- 배달대행사 확인

기존 완료 상태와 진행률 로직을 그대로 사용한다.

세 항목이 모두 완료되었을 때만 최종 마감 버튼을 활성화한다.

실제 변수명은 현재 코드에 맞춘다.

## 미완료 상태

세 항목 중 하나라도 미완료이면 버튼을 비활성화한다.

버튼 위에 남은 항목 수를 표시한다.

예:

마감 완료까지 2개 항목이 남았습니다.

가능하면 실제 남은 개수를 계산해 보여준다.

예:

3개 중 1개 완료
→ 마감 완료까지 2개 항목이 남았습니다.

## 완료 가능 상태

세 항목이 모두 완료되면:

모든 항목을 확인했습니다.

버튼 활성:

7월 15일 마감 완료하기

# 6. 기존 단계 이동 버튼과 역할 구분

현재 화면 최하단 부근에 다음 단계 이동 버튼이 있을 수 있다.

예:

비용 확인하기
배달대행사 확인하기
오늘 마감 완료하기

이 버튼은 현재 다음 미완료 단계로 이동하는 용도다.

이번 Task에서는 다음 단계 이동 버튼을 제거하지 않는다.

역할을 분리한다.

- 단계 이동 버튼: 다음 미완료 입력·확인 화면으로 이동
- 최종 마감 버튼: 모든 단계 확인 후 최종 마감 실행

두 버튼이 연속으로 보여 혼란스럽다면 사이에 충분한 여백과 안내 문구를 둔다.

# 7. 최종 마감 버튼 동작

이번 Task에서는 날짜별 마감 상태 저장소를 구현하지 않는다.

기존 마감 완료 handler가 있다면 그대로 연결한다.

현재 기존 handler가 다음 전역 key를 사용한다면 변경하지 않는다.

closing-completed

중요:

- 새 LocalStorage key 생성 금지
- 날짜별 완료 상태 구현 금지
- 기존 마감 완료 계산 변경 금지
- 새 handler를 중복 구현하지 말고 기존 handler 재사용

# 8. 마감 완료 후 표시

현재 기존 완료 상태가 true라면 버튼 상태를 현재 UX에 맞게 표시한다.

권장:

7월 15일 마감 완료

버튼은 비활성화하거나 완료 상태 카드로 표시한다.

다만 완료 상태가 아직 날짜별이 아니므로, 날짜를 바꾸어도 완료 상태가 유지될 수 있다.

이번 Task에서 이 문제를 해결하지 않는다.
