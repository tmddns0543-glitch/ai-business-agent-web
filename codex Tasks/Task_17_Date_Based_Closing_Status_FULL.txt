AI Business Agent 2세대 웹서비스의 다음 구현 Task를 진행해줘.

이번 Task에서는 마감 상태를 영업일별로 저장하도록 구조를 전환하고, 최종 마감 버튼을 모든 확인 상태와 무관하게 항상 사용할 수 있도록 변경한다.

현재 저장된 완료 상태는 모두 개발용 임시값이므로 기존 전역 완료 key를 마이그레이션하지 않아도 된다.

이번 Task의 핵심 목표는 다음과 같다.

1. 매출 확인 상태를 날짜별로 저장
2. 비용 확인 상태를 날짜별로 저장
3. 배달대행사 확인 상태를 날짜별로 저장
4. 최종 마감 상태를 날짜별로 저장
5. 날짜를 바꾸면 해당 날짜의 진행률과 카드 상태를 복원
6. 매출이 0원이면 과거 완료 상태 때문에 `완료`로 표시되지 않도록 수정
7. 최종 마감 버튼은 미확인 항목이 있어도 항상 활성화
8. 미확인 항목 수는 차단 조건이 아니라 안내 정보로만 표시
9. 완료된 날짜의 데이터를 수정하면 최종 마감 상태만 다시 open으로 변경

## 구현 함수

다음 역할의 함수를 구현한다.

getClosingStatusStorage()

- 전체 저장소 조회
- 잘못된 JSON, 잘못된 version, 잘못된 구조는 빈 저장소 반환
- 서버 환경 안전

getClosingStatusByBusinessDate(businessDate)

- 특정 영업일 상태 조회
- 없으면 기본 상태 반환
- 항상 새 객체 반환

updateClosingStatusByBusinessDate(businessDate, update)

- 해당 날짜의 일부 필드만 갱신
- 다른 날짜 유지
- updatedAt 갱신
- 저장 성공 여부 반환

setSectionConfirmed(businessDate, section)

section:
- "sales"
- "expense"
- "delivery"

역할:
- 해당 섹션 status를 confirmed로 변경
- closingStatus는 그대로 유지
- updatedAt 갱신

setSectionUnconfirmed(businessDate, section)

역할:
- 해당 섹션 status를 unconfirmed로 변경
- closingStatus를 open으로 변경
- completedAt을 null로 변경
- updatedAt 갱신

completeBusinessDayClosing(businessDate)

역할:
- 확인 상태와 무관하게 closingStatus를 completed로 변경
- completedAt 저장
- updatedAt 갱신

reopenBusinessDayClosing(businessDate)

역할:
- closingStatus를 open으로 변경
- completedAt null
- 섹션 확인 상태는 유지
- updatedAt 갱신

필요하다면 helper 이름은 현재 코드에 맞게 조정할 수 있다.

# 3. 오류 처리 원칙

다음 경우 예외 없이 기본 상태 또는 빈 저장소를 반환한다.

- window 없음
- LocalStorage 접근 오류
- key 없음
- 깨진 JSON
- null
- 배열
- 문자열
- version 누락
- version 불일치
- days 누락
- 잘못된 날짜 key
- 잘못된 status 문자열
- completedAt 잘못된 타입
- updatedAt 잘못된 타입

잘못된 기존 값을 임의로 삭제하지 않는다.

입력 객체와 반환 객체를 직접 mutate하지 않는다.

# 4. 기존 전역 완료 key 폐기

다음 전역 key는 더 이상 읽거나 쓰지 않는다.

closing-sales-confirmed
closing-expenses-confirmed
closing-delivery-confirmed
closing-completed

현재 데이터는 테스트용이므로 마이그레이션은 만들지 않는다.

새 날짜별 저장소가 정상 동작하도록 코드 전환 후 위 전역 key 참조를 제거한다.

localStorage.clear() 사용 금지.

자동 removeItem을 제품 코드에 반복적으로 넣지 않는다.

필요하면 개발자가 기존 key를 수동 삭제할 수 있도록 완료 보고에 안내한다.

# 6. 매출 카드 배지 규칙

매출 카드 상태는 선택 영업일 기준으로 계산한다.

규칙:

1. 총매출 0원
   - 배지: 미입력
   - salesStatus가 confirmed여도 완료로 표시하지 않는다.

2. 총매출 > 0원 + salesStatus === unconfirmed
   - 배지: 확인 필요

3. 총매출 > 0원 + salesStatus === confirmed
   - 배지: 완료

즉 과거 날짜에서 완료한 전역 상태가 다른 날짜에 따라오지 않아야 한다.

총매출 0원인데 완료로 보이는 현재 문제를 반드시 해결한다.

# 7. 비용 카드 배지 규칙

비용 기능이 아직 완성되지 않았다면 현재 저장된 비용 합계 또는 기존 임시 값을 그대로 사용할 수 있다.

다만 상태 배지는 날짜별 expenseStatus를 기준으로 표시한다.

권장:

expenseStatus === confirmed
→ 완료

expenseStatus === unconfirmed
→ 확인 필요

향후 비용 데이터가 0원인데 `비용 없음 확인`을 지원하게 되면 confirmed-none 상태로 확장할 수 있다.

이번 Task에서는 status 종류를 unconfirmed / confirmed 두 개만 사용한다.

# 8. 배달대행사 카드 배지 규칙

deliveryStatus === confirmed
→ 완료

deliveryStatus === unconfirmed
→ 확인 필요

현재 배달대행사 상세 데이터가 날짜별 원장이 아니더라도 상태만 날짜별로 전환한다.

# 9. 진행률 계산

진행률은 기존 3단계 기준을 유지한다.

salesStatus === confirmed
→ 1개 완료

expenseStatus === confirmed
→ 1개 완료

deliveryStatus === confirmed
→ 1개 완료

진행률:

0개 → 0%
1개 → 33%
2개 → 67%
3개 → 100%

단, 매출 총매출이 0원이면 salesStatus가 confirmed여도 매출 카드는 완료로 표시하지 않는다.

진행률 계산에서도 매출 0원일 때 sales 완료를 인정할지 아래 규칙을 적용한다.

초기 MVP 권장:

총매출 0원
→ sales 완료로 계산하지 않음

따라서 sales 완료 조건:

salesStatus === confirmed && grossSales > 0

비용·배달대행사는 현재 status만 사용한다.

# 12. 최종 마감 버튼 조건 제거

현재 최종 마감 버튼이 `allCompleted`일 때만 활성화되어 있다면 이 조건을 제거한다.

최종 마감 버튼은 다음 조건에서 항상 활성화한다.

- 화면이 정상 로드됨
- selectedBusinessDate가 유효함
- 저장 처리 중이 아님
- 이미 해당 날짜가 completed 상태가 아님

세부 항목 미확인은 비활성 조건으로 사용하지 않는다.

# 15. 완료된 날짜 데이터 수정 시 처리

플랫폼 매출 저장이 발생할 때 해당 날짜 closingStatus가 completed라면 최종 마감만 reopen한다.

수정 파일:

frontend/src/components/sales/sales-platform-form.tsx

저장 성공 후:

reopenBusinessDayClosing(selectedBusinessDate)

호출.

규칙:

- salesStatus는 유지
- expenseStatus 유지
- deliveryStatus 유지
- closingStatus만 open
- completedAt null

완료된 날짜가 아니어도 호출 가능하도록 idempotent하게 구현할 수 있다.

수정 후 /closing으로 돌아왔을 때:

수정사항이 반영되었습니다.
마감을 다시 완료할 수 있습니다.

같은 안내를 넣을 수 있지만 과도한 UI 추가는 하지 않는다.

# 16. 신규 입력 시 처리

새로운 날짜에 매출을 처음 저장하면:

- closingStatus는 open 유지
- salesStatus는 자동 confirmed로 만들지 않는다

사용자는 `/closing/sales`에서 `매출 확인 완료`를 눌러야 한다.

즉:

입력 저장 ≠ 확인 완료

원칙을 유지한다.
