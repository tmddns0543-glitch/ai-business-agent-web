# AI Business Agent Web UI Wireframes

## 1. 문서 목적

이 문서는 AI Business Agent Web의 주요 화면 구조와 사용자 행동을 정의한다.

초기 설계 기준은 모바일 화면이다.

PC 화면에서는 모바일 구조를 유지하면서 폭만 확장한다.

---

# 2. 공통 UI 원칙

## 2.1 모바일 우선

- 주요 버튼은 한 손으로 누를 수 있는 크기로 만든다.
- 화면 하단에 핵심 메뉴를 고정한다.
- 금액 입력 시 숫자 키패드를 사용한다.
- 한 화면에 하나의 핵심 행동만 강조한다.

## 2.2 쉬운 표현

전문 용어보다 점주가 이해하기 쉬운 표현을 사용한다.

예시:

- 예상 플랫폼 공제액
- 실제로 들어올 예상 금액
- 이번 달 예상 이익
- 오늘 정산

## 2.3 상태를 명확하게 표시

사용자는 현재 입력 상태를 바로 이해할 수 있어야 한다.

상태 예시:

- 입력 전
- 작성 중
- 입력 완료
- 수정 필요

## 2.4 저장 전 확인

중요 입력은 저장 전에 요약 화면을 제공한다.

## 2.5 수정 가능성

사용자는 저장한 값을 쉽게 다시 열고 수정할 수 있어야 한다.

---

# 3. 모바일 하단 메뉴

모바일 하단에는 다음 5개 메뉴를 고정한다.

1. 홈
2. 매출입력
3. 비용입력
4. 경영성과
5. 더보기

현재 선택된 메뉴는 강조 표시한다.

---

# 4. 홈 화면

## 4.1 화면 목적

홈 화면은 사용자가 다음 세 가지를 즉시 이해하도록 한다.

1. 오늘 입력을 완료했는가
2. 이번 달 얼마를 팔았는가
3. 실제로 얼마가 남을 것으로 예상되는가

## 4.2 화면 상단

표시 정보:

- 사업장명
- 현재 날짜
- 사용자 인사말

예시:

안녕하세요.
떴다파닭 진주점 사장님

2026년 7월 12일 일요일

## 4.3 오늘 정산 상태

카드 형태로 표시한다.

### 오늘 매출

상태:

- 입력 전
- 작성 중
- 입력 완료

버튼:

- 오늘 매출 입력하기
- 오늘 매출 수정하기

### 오늘 비용

상태:

- 확인 전
- 입력 완료
- 발생 비용 없음

버튼:

- 오늘 비용 확인하기
- 오늘 비용 수정하기

## 4.4 핵심 행동 버튼

오늘 매출이 입력되지 않은 경우 가장 큰 버튼을 표시한다.

버튼 문구:

오늘 정산 시작하기

오늘 매출이 이미 입력된 경우:

오늘 정산 확인하기

## 4.5 이번 달 핵심 숫자

다음 4개를 우선 표시한다.

- 이번 달 총매출
- 예상 플랫폼 수수료
- 예상 정산금액
- 월말 예상 영업이익

예시:

이번 달 총매출
18,420,000원

예상 플랫폼 수수료
1,284,000원

예상 정산금액
17,136,000원

월말 예상 영업이익
2,840,000원

## 4.6 최근 흐름

초기 MVP에서는 다음 두 개만 표시한다.

- 최근 7일 매출 그래프
- 플랫폼별 매출 비중

## 4.7 홈 화면 주요 행동

- 오늘 정산 시작
- 오늘 매출 수정
- 오늘 비용 입력
- 경영성과 이동

---

# 5. 가이드형 매출입력

## 5.1 화면 목적

처음 사용하는 점주도 설명 없이 매출입력을 완료할 수 있게 한다.

한 화면에서 하나의 질문만 보여준다.

## 5.2 진입 화면

제목:

오늘 하루 정산을 시작해볼까요?

표시 정보:

- 정산 날짜
- 예상 소요시간
- 활성화된 매출채널 수

예시:

2026년 7월 12일

약 3분 정도 걸려요.

오늘 입력할 채널:
배달의민족, 쿠팡이츠, 일반 카드, 현금

버튼:

정산 시작하기

보조 버튼:

빠른 입력으로 전환

## 5.3 진행 상태

화면 상단에 진행률을 표시한다.

예시:

배달의민족 2 / 5

또는 진행바:

40% 완료

## 5.4 질문 화면 구조

구성:

- 이전 버튼
- 현재 채널명
- 질문 문구
- 입력창
- 0원 버튼
- 다음 버튼
- 진행률

예시:

배달의민족

오늘 배달의민족 선결제 매출은 얼마인가요?

[ 420,000 ]

[0원] [다음]

## 5.5 금액 입력 규칙

- 숫자 키패드를 사용한다.
- 입력 즉시 천 단위 구분기호를 표시한다.
- 빈값은 저장하지 않는다.
- 0원은 별도 버튼으로 빠르게 입력한다.
- 음수는 허용하지 않는다.

## 5.6 주문 수 입력

매출 입력 후 필요한 채널만 주문 수를 묻는다.

예시:

오늘 배민원 주문은 몇 건이었나요?

[ 14 ]

설명:

주문당 평균매출과 수수료 분석에 사용됩니다.

## 5.7 채널별 질문 예시

### 배달의민족

- 선결제 매출
- 카드 매출
- 현금 매출
- 배민원 매출
- 배민원 주문 수

### 요기요

- 선결제 매출
- 카드 매출
- 현금 매출
- 요기배달 매출
- 요기배달 주문 수

### 쿠팡이츠

- 쿠팡이츠 매출
- 주문 수

### 땡겨요

- 선결제 매출

### 일반 매출

- 일반 카드
- 현금
- 계좌이체

사용하지 않는 채널은 질문하지 않는다.

## 5.8 자동 임시저장

각 단계에서 다음 버튼을 누를 때 임시저장한다.

사용자가 중간에 나가더라도 작성 중 상태로 보존한다.

다시 들어오면 마지막 단계부터 이어서 작성할 수 있다.

## 5.9 이전 단계 이동

사용자는 이전 버튼으로 앞 단계로 돌아갈 수 있다.

기존 입력값은 유지된다.

## 5.10 중간 종료

중간에 나가는 경우 다음 안내를 표시한다.

지금까지 입력한 내용은 임시저장되었습니다.

나중에 이어서 입력할 수 있습니다.

# 6. 매출입력 요약 화면

## 6.1 화면 목적

저장 전에 입력한 내용을 확인하고 수정할 수 있게 한다.

## 6.2 표시 내용

플랫폼별 입력내역:

배달의민족
- 선결제 420,000원
- 카드 85,000원
- 현금 30,000원
- 배민원 210,000원
- 배민원 주문 14건

쿠팡이츠
- 매출 180,000원
- 주문 9건

일반 매출
- 카드 120,000원
- 현금 50,000원

## 6.3 합계

- 총매출
- 예상 플랫폼 수수료
- 예상 정산금액

예시:

총매출
1,095,000원

예상 플랫폼 수수료
82,500원

예상 정산금액
1,012,500원

## 6.4 주요 행동

- 수정하기
- 저장하고 정산 완료

---

# 7. 매출입력 완료 화면

## 7.1 화면 목적

입력의 결과를 즉시 보여준다.

## 7.2 완료 메시지

오늘 정산이 완료되었습니다.

## 7.3 핵심 결과

- 오늘 총매출
- 예상 플랫폼 수수료
- 예상 정산금액
- 플랫폼별 예상 정산금액

## 7.4 다음 행동

- 오늘 비용 입력하기
- 홈으로 이동
- 매출 다시 수정하기

---

# 8. 빠른 매출입력

## 8.1 화면 목적

입력에 익숙한 사용자가 한 화면에서 빠르게 입력하게 한다.

## 8.2 화면 구조

날짜 선택

플랫폼별 접이식 영역:

- 배달의민족
- 요기요
- 쿠팡이츠
- 땡겨요
- 일반 매출

각 영역 안에서 활성화된 항목만 보여준다.

## 8.3 주요 행동

- 임시저장
- 입력 요약
- 저장 완료
- 가이드 입력으로 전환

---

# 9. 빈 상태 화면

데이터가 없는 사용자는 차트 대신 안내 화면을 본다.

예시:

아직 입력된 매출이 없습니다.

첫 매출을 입력하면
이번 달 매출과 예상 정산금액을 확인할 수 있어요.

버튼:

첫 매출 입력하기

---

# 10. 오류 상태

## 10.1 네트워크 오류

입력한 내용은 기기에 임시저장한다.

안내:

인터넷 연결이 불안정합니다.
연결이 복구되면 다시 저장할 수 있습니다.

## 10.2 저장 실패

안내:

저장하지 못했습니다.
입력한 내용은 유지되고 있습니다.

버튼:

다시 저장하기

## 10.3 중복 입력

같은 날짜의 매출이 이미 존재하면:

이미 입력된 매출이 있습니다.

기존 내역을 수정하시겠어요?

버튼:

- 기존 내역 보기
- 수정하기
- 취소
