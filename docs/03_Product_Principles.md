# AI Business Agent Web Product Principles

## 1. 제품의 핵심 목적

AI Business Agent Web은 외식업 점주가 매일 짧은 시간 동안 매출과 비용을 입력하고,
플랫폼 수수료, 예상 정산금액, 월별 손익을 쉽게 이해할 수 있도록 돕는 서비스다.

초기 웹 MVP는 AI 기능보다 정확한 입력, 정산, 손익 계산과 쉬운 사용 경험을 우선한다.

## 2. 핵심 사용자 행동

사용자는 다음 행동을 반복한다.

1. 매일 매출을 입력한다.
2. 발생한 비용을 입력한다.
3. 플랫폼 수수료와 예상 정산금액을 확인한다.
4. 월별 경영성과를 확인한다.

## 3. 핵심 가치

- 여러 플랫폼의 매출을 한곳에서 입력할 수 있다.
- 플랫폼별 예상 수수료를 자동으로 계산한다.
- 실제로 입금될 것으로 예상되는 금액을 확인할 수 있다.
- 매출과 비용을 합쳐 월별 손익을 확인할 수 있다.
- 회계 지식이 없어도 이해할 수 있다.

## 4. UX 원칙

### 4.1 모바일 우선

모든 핵심 기능은 휴대폰에서 편하게 사용할 수 있어야 한다.

### 4.2 한 화면, 한 행동

가이드 입력에서는 한 화면에 하나의 핵심 질문만 보여준다.

### 4.3 불필요한 항목 제거

사용자가 설정에서 활성화한 매출채널과 비용항목만 보여준다.

### 4.4 입력 즉시 결과 제공

매출 입력 완료 직후 다음 정보를 보여준다.

- 총매출
- 예상 플랫폼 수수료
- 예상 정산금액
- 플랫폼별 정산금액

### 4.5 초보자와 숙련자 모두 지원

- 처음 사용자는 가이드 입력을 이용한다.
- 익숙한 사용자는 빠른 입력을 이용한다.
- 최근 사용한 입력 방식을 기억한다.

### 4.6 수정 가능성

사용자가 잘못 입력한 데이터를 쉽게 확인하고 수정할 수 있어야 한다.

### 4.7 전문 용어 최소화

회계와 플랫폼 정산 용어는 쉬운 표현을 우선하고,
필요한 경우 설명을 함께 제공한다.

## 5. 핵심 화면

- 서비스 소개
- 회원가입
- 로그인
- 사업장 초기설정
- 홈
- 가이드 매출입력
- 빠른 매출입력
- 비용입력
- 입력 완료 결과
- 배달대행사 현황
- 경영성과
- 설정
- 피드백

## 6. 초기 웹 MVP에서 하지 않는 것

- AI Morning Brief
- 사업 건강도
- CFO Agent
- COO Agent
- 자동 발주
- POS 연동
- 은행 계좌 연동
- 구독 결제
- 네이티브 앱
- 프랜차이즈 본사 관리

## 7. 기능 추가 판단 기준

새 기능은 다음 질문을 모두 검토한 후 추가한다.

1. 점주의 입력시간을 줄이는가?
2. 점주가 돈의 흐름을 더 쉽게 이해하게 하는가?
3. 반복적으로 사용할 가능성이 있는가?
4. 기존 화면을 복잡하게 만들지 않는가?
5. 실제 베타 사용자의 요구가 확인되었는가?
