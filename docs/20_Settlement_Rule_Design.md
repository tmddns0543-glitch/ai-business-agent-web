# AI Business Agent Web Settlement Rule Design

## 1. 문서 목적

이 문서는 AI Business Agent Web에서 사용하는
플랫폼별 예상 공제액과 예상 정산금액의 계산 기준을 정의한다.

이 문서는 다음 기능의 공통 기준으로 사용한다.

- 내 가게 설정
- 플랫폼별 매출 입력
- 예상 공제액 계산
- 예상 정산금액 계산
- 마감 결과
- 월별 경영성과
- 향후 CFO 재무 분석

수수료 계산은 플랫폼 전체가 아니라
플랫폼 안의 주문 유형과 결제 방식별로 구분한다.

---

# 2. 핵심 용어

## 2.1 총매출

점주가 입력한 모든 매출의 합계다.

```text
총매출
= 배달의민족 매출
+ 쿠팡이츠 매출
+ 요기요 매출
+ 땡겨요 매출
+ 일반결제 매출
```

---

## 2.2 비율 수수료

매출금액에 수수료율을 곱하여 계산하는 수수료다.

예시:

- 중개수수료
- 결제수수료
- 카드수수료

```text
비율 수수료
= 적용 매출 × 수수료율
```

---

## 2.3 건당 비용

주문 수에 건당 비용을 곱하여 계산한다.

예시:

- 배민원 건당 배달료
- 쿠팡이츠 건당 배달료
- 요기배달 건당 배달료

```text
건당 배달료
= 주문 수 × 가게별 건당 배달료
```

---

## 2.4 수수료 부가세

부가세 대상 수수료와 건당 비용 합계에
10%를 적용한다.

```text
수수료 부가세
= 부가세 대상 공제항목 합계 × 10%
```

부가세는 매출 전체에 적용하지 않는다.

---

## 2.5 예상 공제액

플랫폼이 매출에서 공제할 것으로 예상되는 금액이다.

```text
예상 공제액
= 중개수수료
+ 결제수수료
+ 카드수수료
+ 건당 배달료
+ 수수료 부가세
```

적용되지 않는 항목은 0원으로 계산한다.

---

## 2.6 예상 정산금액

```text
예상 정산금액
= 총매출 - 예상 공제액
```

예상 정산금액은 실제 입금액이 아니다.

플랫폼 정책, 프로모션, 할인 분담, 광고비, 정산 조정,
환불 및 기타 공제에 따라 실제 금액과 차이가 발생할 수 있다.

---

# 3. 설계 원칙

## 3.1 설정값 기반 계산

모든 수수료는 사업장별 설정값을 기준으로 계산한다.

기본 수수료율은 추천값으로 제공할 수 있지만,
점주는 실제 계약 조건에 맞게 수정할 수 있어야 한다.

---

## 3.2 플랫폼이 아니라 매출항목별 계산

같은 플랫폼 안에서도 주문 유형과 결제 방식에 따라
계산 규칙이 다르다.

예시:

배달의민족 가게배달 선결제와
배달의민족 가게배달 카드결제는
같은 계산식을 사용하지 않는다.

---

## 3.3 결제수수료와 카드수수료 중복 금지

배달의민족과 요기요의 가게배달 카드결제에는
일반 결제수수료를 적용하지 않는다.

대신 카드수수료를 적용한다.

```text
중개수수료: 적용
일반 결제수수료: 미적용
카드수수료: 적용
```

결제수수료와 카드수수료를 동시에 적용하면 안 된다.

---

## 3.4 주문 수는 매출에 합산하지 않는다

주문 수는 건당 배달료 계산과
주문당 평균매출 분석에만 사용한다.

```text
총매출에 주문 수를 더하지 않는다.
```

---

## 3.5 부가세는 공제항목 기준으로 계산

부가세는 각 수수료와 배달료를 계산한 뒤 적용한다.

수수료율에 부가세를 미리 포함하여 계산하지 않는다.

---

## 3.6 계산 결과의 성격

현재 Web MVP에서 계산하는 금액은 예상값이다.

사용자 화면에서는 다음 용어를 사용한다.

- 예상 공제액
- 예상 정산금액

확정 공제액이나 확정 정산금액으로 표시하지 않는다.

---

# 4. 내 가게 설정

## 4.1 접근 경로

```text
더보기
→ 내 가게 설정
→ 매출채널 설정
```

---

## 4.2 설정 방식

플랫폼별 카드에서 세부 설정으로 이동한다.

```text
배달의민족
쿠팡이츠
요기요
땡겨요
일반결제
```

---

## 4.3 공통 설정 항목

필요에 따라 다음 항목을 설정한다.

- 중개수수료율
- 결제수수료율
- 카드수수료율
- 건당 배달료
- 수수료 부가세 적용 여부
- 부가세율
- 사용 여부

---

## 4.4 기본 추천값

최초 설정 시 일반적인 기준값을 제공한다.

단, 다음 문구를 함께 표시한다.

```text
플랫폼 계약과 가게 조건에 따라 실제 수수료가 다를 수 있습니다.
실제 계약 내용을 확인하고 수정해주세요.
```

---

## 4.5 비율 입력 단위

사용자는 화면에서 퍼센트 단위로 입력한다.

예시:

```text
6.8
```

화면 표시:

```text
6.8%
```

내부 계산 시:

```text
0.068
```

로 변환한다.

# 5. 배달의민족 정산 규칙

## 5.1 배달의민족 입력 항목

현재 매출 입력 항목:

- 가게배달 선결제
- 가게배달 카드
- 가게배달 현금
- 배민원 매출
- 배민원 주문 수

향후 필요 시 픽업 매출 항목을 별도로 추가한다.

---

## 5.2 가게배달 선결제

기본 설정값:

```text
중개수수료율: 6.8%
결제수수료율: 3.0%
부가세율: 10%
```

계산식:

```text
중개수수료
= 선결제 매출 × 중개수수료율

결제수수료
= 선결제 매출 × 결제수수료율

수수료 부가세
= (중개수수료 + 결제수수료) × 10%

예상 공제액
= 중개수수료
+ 결제수수료
+ 수수료 부가세
```

---

## 5.3 가게배달 카드결제

중요 예외 규칙:

```text
중개수수료: 적용
일반 결제수수료 3%: 미적용
카드수수료: 적용
```

기본 설정값:

```text
중개수수료율: 6.8%
카드수수료율: 최대 1.5%
부가세율: 10%
```

계산식:

```text
중개수수료
= 카드 매출 × 중개수수료율

카드수수료
= 카드 매출 × 카드수수료율

수수료 부가세
= (중개수수료 + 카드수수료) × 10%

예상 공제액
= 중개수수료
+ 카드수수료
+ 수수료 부가세
```

일반 결제수수료 3%를 추가하면 안 된다.

---

## 5.4 가게배달 현금결제

기본 설정값:

현금결제는 플랫폼 결제수수료를 적용하지 않는다.

---

## 5.5 배민원

기본 설정값:

```text
중개수수료율: 7.8%
결제수수료율: 3.0%

부가세율: 10%
```

건당 배달료는 가게별로 다르므로
내 가게 설정에서 수정 가능해야 한다.

= 배민원 주문 수 × 건당 배달료

+ 결제수수료
+ 배달료
+ 수수료 부가세
```

쿠팡이츠 매출 입력 항목을 배달과 픽업으로 분리해야 한다.

```

계산식:

```text
중개수수료
= 요기포장 매출 × 중개수수료율

결제수수료
= 요기포장 매출 × 결제수수료율

수수료 부가세
= (중개수수료 + 결제수수료) × 10%

예상 공제액
= 중개수수료
+ 결제수수료
+ 수수료 부가세
```

요기포장은 건당 배달료를 적용하지 않는다.

현재 매출 입력 화면에 요기포장 항목이 없다면
포장 정산 구현 전에 입력 항목을 추가해야 한다.

## 7.7 요기패스 추가 수수료

요기패스 회원 주문에는
중개수수료 1.2%가 추가로 부과될 수 있다.

그러나 가맹점이 입력 시점에
요기패스 고객과 일반 고객을 구분할 수 없다.

따라서 초기 Web MVP에서는
요기패스 추가 수수료를 자동 계산하지 않는다.

내 가게 설정에서 다음 방식 중 하나를 지원할 수 있다.

### 방식 A: 계산 제외

```text
요기패스 추가 수수료 반영 안 함
```

### 방식 B: 전체 요기요 매출에 추정 적용

```text
요기패스 추가 수수료율: 1.2%
추정 적용 비율: 사용자가 입력
```

예시:

```text
요기패스 주문 추정 비율: 30%
```

계산식:

```text
요기패스 추정 추가수수료
= 요기요 대상 매출
× 요기패스 추가수수료율
× 요기패스 추정 적용 비율
```

초기 MVP 기본값은 계산 제외로 한다.

사용자에게 실제 정산금액과 차이가 발생할 수 있음을 안내한다.

# 11. 계산 순서

각 플랫폼과 매출항목은 다음 순서로 계산한다.

```text
1. 매출 원본값 조회
2. 해당 매출항목의 정산 규칙 조회
3. 중개수수료 계산
4. 결제수수료 또는 카드수수료 계산
5. 주문 수 기반 건당 비용 계산
6. 부가세 대상 항목 합산
7. 부가세 계산
8. 예상 공제액 계산
9. 예상 정산금액 계산
```

카드결제 예외에서는
결제수수료와 카드수수료 중 하나만 적용한다.

# 18. 성공 기준

- 점주가 내 가게 설정에서 실제 계약 수수료를 수정할 수 있다.
- 플랫폼별 매출 입력값과 주문 수를 기준으로 예상 공제액이 계산된다.
- 결제수수료와 카드수수료가 중복 적용되지 않는다.
- 배달의민족과 요기요 카드결제 예외 규칙이 정확히 작동한다.
- 주문 수는 매출에 포함되지 않고 건당 비용 계산에만 사용된다.
- 부가세가 지정된 공제항목 합계에만 적용된다.
- 플랫폼별 예상 정산금액과 전체 예상 정산금액이 일치한다.
- 계산 로직이 화면 파일에 중복되지 않는다.
- 실제 계약 조건에 따라 사용자가 기본값을 수정할 수 있다.
- 향후 FastAPI와 PostgreSQL로 이전 가능한 구조를 유지한다.
