# AI Business Agent Web Interaction Design

## 1. 문서 목적

이 문서는 AI Business Agent Web의 사용자 인터랙션(Interaction)을 정의한다.

화면을 예쁘게 만드는 것이 목적이 아니다.

사용자가

"편하다"

"빠르다"

"생각할 것이 없다"

라고 느끼게 만드는 것이 목적이다.

---

# 2. Interaction 철학

AI Business Agent는

입력 프로그램이 아니다.

사용자의 하루를 정리하는 프로그램이다.

모든 인터랙션은

"정산을 끝냈다."

라는 만족감을 주는 방향으로 설계한다.

---

# 3. 버튼 규칙

## Primary Button

항상

현재 화면에서

가장 중요한 행동

한 개만 Primary Button으로 표시한다.

예시

오늘 정산 시작하기

저장하기

회원가입

비용 추가

Primary Button은

한 화면에 하나만 존재한다.

---

## Secondary Button

뒤로가기

취소

수정

다시보기

등은 Secondary Button으로 표시한다.

Primary보다 시각적으로 약해야 한다.

---

# 4. 저장 버튼

저장 버튼은

누르는 순간

즉시 로딩 상태로 변경한다.

예시

저장하기

↓

저장중...

↓

✔ 저장 완료

사용자는

버튼을 여러 번 누를 필요가 없어야 한다.

---

# 5. 입력 완료

입력이 완료되면

반드시

완료 화면을 보여준다.

예시

🎉

오늘 정산이 완료되었습니다.

오늘 총매출

745,000원

예상 정산금

682,600원

[ 비용 입력 ]

[ Home ]

입력 후

바로 Home으로 이동하지 않는다.

---

# 6. 로딩

모든 저장은

즉시 로딩 상태를 보여준다.

예시

...

저장 중입니다.

잠시만 기다려주세요.

로딩이 1초 이하라도

사용자가 저장되고 있다는 것을

느껴야 한다.

---

# 7. 애니메이션

애니메이션은

짧고 자연스럽게 사용한다.

사용 목적

- 화면 전환
- 저장 완료
- 카드 펼침
- 그래프 등장

과도한 애니메이션은 사용하지 않는다.

---

# 8. 카드(Card)

모든 핵심 정보는

Card 형태로 표현한다.

예시

오늘 매출

이번달 매출

예상 정산금

영업이익

Card는

터치하면

세부 화면으로 이동한다.

---

# 9. 입력창

입력창은

항상

입력 목적을 먼저 설명한다.

좋은 예

오늘 배달의민족 선결제 매출은 얼마인가요?

나쁜 예

선결제

입력창은

숫자 입력 시

즉시

천단위 구분기호를 표시한다.

예

420000

↓

420,000원

---

# 10. 진행률

가이드 입력에서는

현재 진행 상태를

항상 보여준다.

예시

■■■■□□□□

40%

2 / 5

사용자는

언제 끝나는지

예측할 수 있어야 한다.

---

# 11. 오류

오류는

사용자를 탓하지 않는다.

좋은 예

저장하지 못했습니다.

입력한 내용은 유지되고 있습니다.

다시 저장해주세요.

나쁜 예

오류가 발생했습니다.

---

# 12. 확인 메시지

삭제

초기화

복원

등

되돌릴 수 없는 행동은

반드시

한 번 더 확인한다.

예시

정말 삭제하시겠습니까?

[ 삭제 ]

[ 취소 ]

---

# 13. 빈 화면

데이터가 없을 때는

빈 화면을 그대로 보여주지 않는다.

예시

아직 입력된 매출이 없습니다.

오늘 첫 매출을 입력해보세요.

[ 매출 입력 ]

---

# 14. 성공 경험

사용자는

작은 성공을 자주 느껴야 한다.

예시

✓ 오늘 매출 입력 완료

✓ 오늘 비용 입력 완료

✓ 오늘 정산 완료

체크가 하나씩 늘어나면서

성취감을 느끼게 한다.

---

# 15. Home Interaction

Home은

사용자가

무엇을 해야 하는지

생각하지 않도록 만든다.

Home에서

가장 큰 버튼은

오늘 해야 하는 일

이다.

예시

오늘 정산 시작하기

↓

매출 입력

↓

비용 입력

↓

정산 완료

↓

Home

---

# 16. 입력 흐름

입력은

사용자를 시험하는 과정이 아니다.

프로그램이

사용자를 안내하는 과정이다.

예시

오늘 배달의민족 선결제 매출은 얼마인가요?

↓

입력

↓

다음

↓

오늘 배달의민족 카드 매출은 얼마인가요?

사용자는

무엇을 입력해야 하는지

고민하면 안 된다.

---

# 17. 피드백

사용자는

언제든지

불편한 점을 보낼 수 있어야 한다.

피드백 버튼은

항상

더보기 메뉴에서 접근 가능하다.

---

# 18. 알림

필요한 경우에만 알림을 사용한다.

좋은 예

오늘 정산이 아직 완료되지 않았습니다.

나쁜 예

불필요한 광고성 알림

---

# 19. 감정

사용자가

이 프로그램을 사용할 때

느껴야 하는 감정

- 편안함

- 신뢰

- 단순함

- 성취감

느껴서는 안 되는 감정

- 복잡함

- 불안

- 귀찮음

- 피로감

---

# 20. 최종 UX 목표

사용자는

프로그램을 사용하면서

다음과 같이 느껴야 한다.

"오늘 정산도 금방 끝났네."

"생각보다 쉽다."

"이제 엑셀 안 봐도 되겠다."

"매일 쓰는 게 습관이 되겠다."
