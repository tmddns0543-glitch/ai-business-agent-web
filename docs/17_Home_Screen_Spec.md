# AI Business Agent Web Home Screen Specification

## 1. 문서 목적

이 문서는 AI Business Agent Web의 모바일 Home 화면을 실제로 구현할 수 있도록 상세 구조와 상태별 동작을 정의한다.

Home은 단순한 통계 대시보드가 아니다.

Home은 사용자가 다음 순서로 사업 상태를 이해하도록 돕는다.

1. 마지막 완료 영업일의 성적 확인
2. 이번 달 누적 성적 확인
3. 현재 처리해야 할 마감 확인
4. 다음 행동 실행

사용자에게 보이는 용어는 `정산`보다 `마감`을 사용한다.

내부 시스템에서는 다음 용어를 사용할 수 있다.

- business_day
- closing_status
- closing_completed_at

---

# 2. Home 핵심 사용자 흐름

사용자가 Home에 진입하면 프로그램은 다음 순서로 상태를 판단한다.

1. 현재 기준 영업일을 계산한다.
2. 미완료 영업일이 있는지 확인한다.
3. 마지막 완료 영업일을 확인한다.
4. 이번 달 누적 성과를 계산한다.
5. 가장 필요한 행동을 Primary Button으로 표시한다.

Home의 정보 순서는 다음과 같다.

마지막 완료 영업일 성적

↓

이번 달 누적 성적

↓

현재 마감 상태와 해야 할 일

↓

최근 흐름

---

# 3. 영업일 판단 기준

## 3.1 기본 설정

사업장 설정에서 다음 정보를 관리한다.

- 영업 종료 시간
- 휴무일
- 시간대
- 선택적으로 다음 날 마감 여부

예시:

영업 종료 시간: 23:00  
시간대: Asia/Seoul

## 3.2 현재 기준 영업일

현재 시간이 영업 종료 시간 이전이라면,
완료해야 할 가장 최근 영업일을 기준으로 판단한다.

현재 시간이 영업 종료 시간 이후라면,
해당 날짜의 마감을 진행할 수 있다.

단순히 현재 날짜만으로 판단하지 않고,
저장된 마감 상태와 영업 종료 시간을 함께 사용한다.

## 3.3 미완료 영업일 우선

현재 날짜가 2026년 7월 13일이고 다음 상태라면:

- 2026년 7월 11일 마감 완료
- 2026년 7월 12일 마감 미완료

Home은 2026년 7월 12일의 미완료 상태를 가장 먼저 안내한다.

표시 예시:

2026년 7월 12일 마감이 아직 끝나지 않았어요.

버튼:

[ 7월 12일 마감 시작하기 ]

# 4. Home 전체 화면 구조

모바일 Home은 다음 영역으로 구성한다.

1. 상단 헤더
2. 마지막 완료 영업일 성적
3. 이번 달 누적 성적
4. 마감 상태 및 행동 카드
5. 최근 흐름
6. 하단 고정 메뉴

---

# 5. 상단 헤더

## 5.1 표시 정보

- 사업장명
- 사용자 인사말
- 알림 버튼
- 프로필 또는 더보기 버튼

예시:

좋은 아침이에요.  
떴다파닭 진주점

## 5.2 시간대별 문구

오전:

좋은 아침이에요.

오후:

오늘도 힘내세요.

영업 종료 이후:

오늘 장사도 수고하셨어요.

문구는 장식적인 AI 문장보다 짧고 자연스럽게 유지한다.

---

# 6. 마지막 완료 영업일 성적 카드

## 6.1 목적

사용자가 가장 최근에 마감한 하루의 결과를 빠르게 확인한다.

## 6.2 카드 제목

최근 마감 결과

또는 날짜에 따라:

어제 마감 결과

단, 실제 마지막 완료 영업일이 어제가 아니라면 `어제`라고 표시하지 않는다.

예시:

7월 11일 마감 결과

## 6.3 핵심 표시 항목

- 총매출
- 예상 플랫폼 공제액
- 예상 정산금액
- 배달대행비
- 등록 비용 합계

초기 화면에서는 가장 중요한 숫자 두 개를 크게 표시한다.

예시:

총매출  
745,000원

예상 정산금액  
682,600원

나머지 정보는 작게 표시하거나 상세보기에서 확인한다.

## 6.4 카드 행동

카드를 누르면 해당 영업일의 마감 결과 상세화면으로 이동한다.

버튼:

[ 마감 결과 보기 ]

---

# 7. 이번 달 누적 성적 카드

## 7.1 목적

사용자가 이번 달 전체 흐름을 한눈에 이해한다.

## 7.2 표시 항목

- 이번 달 총매출
- 예상 플랫폼 공제액
- 예상 정산금액
- 누적 비용
- 예상 영업이익
- 영업이익률

Home에서는 핵심 숫자를 최대 3개만 강조한다.

추천 우선순위:

1. 이번 달 총매출
2. 예상 정산금액
3. 예상 영업이익

## 7.3 데이터 기준

이번 달 누적 성적은 저장 완료된 원본 데이터를 기준으로 계산한다.

매출은 저장 완료된 영업일 데이터를 반영한다.

비용은 저장된 거래를 즉시 반영한다.

미완료 매출 초안은 월 성과에 반영하지 않는다.

## 7.4 월말 예상

충분한 과거 데이터가 존재하면 월말 예상값을 표시할 수 있다.

과거 데이터가 부족하다면 억지로 예상값을 표시하지 않는다.

대체 문구:

월말 예상치를 계산하려면 조금 더 많은 기록이 필요해요.

## 7.5 카드 행동

카드를 누르면 경영성과 화면의 현재 월로 이동한다.

버튼:

[ 이번 달 자세히 보기 ]

---

# 8. 마감 상태 카드

## 8.1 목적

사용자가 현재 완료해야 할 마감을 즉시 이해하고 실행한다.

## 8.2 상태 종류

마감 상태는 다음과 같이 구분한다.

- not_started: 시작 전
- in_progress: 작성 중
- completed: 완료
- overdue: 이전 영업일 미완료

## 8.3 시작 전

표시 예시:

7월 12일 마감이 아직 시작되지 않았어요.

Primary Button:

[ 마감 시작하기 ]

## 8.4 작성 중

표시 예시:

7월 12일 마감을 작성 중이에요.

현재 단계:

배달의민족 매출 입력

Primary Button:

[ 이어서 마감하기 ]

Secondary Button:

[ 작성 내용 삭제 ]

## 8.5 완료

표시 예시:

7월 12일 마감 완료

오늘도 수고하셨어요.

Primary Button:

[ 마감 결과 확인하기 ]

## 8.6 이전 영업일 미완료

미완료 영업일이 여러 개라면 가장 오래된 미완료 영업일부터 처리하도록 안내한다.

예시:

완료되지 않은 마감이 2건 있어요.

- 7월 11일
- 7월 12일

Primary Button:

[ 7월 11일 마감하기 ]

Secondary Button:

[ 미완료 마감 전체 보기 ]

---

# 9. 마감 단계 구성

초기 Web MVP의 마감은 다음 단계로 구성한다.

1. 매출 입력
2. 비용 확인
3. 배달대행사 거래 확인
4. 마감 결과 확인
5. 마감 완료

비용은 모든 거래를 마감 과정에서 새로 입력하는 것이 아니다.

이미 저장한 비용 거래를 확인하고,
누락된 비용이 있으면 추가한다.

## 9.1 단계별 상태

각 단계는 다음 상태를 가진다.

- 대기
- 작성 중
- 완료
- 확인 필요

예시:

매출 입력  
완료

비용 확인  
3건 / 455,500원

배달대행사  
확인 필요

---

# 10. 최근 흐름

## 10.1 초기 차트

Home 하단에서는 다음 두 개만 제공한다.

- 최근 7개 완료 영업일 매출
- 플랫폼별 매출 비중

## 10.2 표시 원칙

차트는 숫자를 대신하지 않는다.

먼저 핵심 숫자를 표시하고,
차트는 변화와 비중을 이해하는 보조 수단으로 사용한다.

## 10.3 데이터 없는 경우

표시 예시:

아직 비교할 매출 기록이 부족해요.

마감을 2일 이상 완료하면 최근 흐름을 확인할 수 있어요.

---

# 11. Home 상태별 화면

## 11.1 신규 사용자

아직 완료된 마감이 없다.

표시 순서:

- 환영 안내
- 첫 마감 시작 카드
- 첫 입력으로 확인할 수 있는 결과 안내

예시:

첫 마감을 시작해보세요.

매출을 입력하면 예상 플랫폼 공제액과 정산금액을 바로 확인할 수 있어요.

버튼:

[ 첫 마감 시작하기 ]

이번 달 성적 카드와 차트는 빈 상태로 표시한다.

---

## 11.2 마지막 마감 완료, 현재 마감 시작 전

표시 순서:

- 최근 마감 결과
- 이번 달 성적
- 현재 영업일 마감 시작
- 최근 흐름

---

## 11.3 마감 작성 중

표시 순서:

- 작성 중인 마감 카드
- 이어서 마감하기 버튼
- 최근 완료 마감 결과
- 이번 달 성적

작성 중 마감이 가장 중요한 행동이므로 상단 우선순위를 높인다.

---

## 11.4 마감 지연

표시 순서:

- 미완료 마감 경고 카드
- 마감 시작 버튼
- 마지막 완료 영업일 결과
- 이번 달 성적

경고는 위협적인 빨간색보다 주의가 필요한 상태로 표현한다.

사용자를 비난하는 문구는 사용하지 않는다.

나쁜 예:

마감하지 않았습니다.

좋은 예:

7월 12일 마감이 아직 남아 있어요.

---

## 11.5 모든 마감 완료

표시 순서:

- 최근 마감 결과
- 이번 달 성적
- 완료 메시지
- 최근 흐름

완료 메시지 예시:

현재까지 필요한 마감을 모두 끝냈어요.

---

# 12. Primary Button 규칙

Home에는 가장 중요한 Primary Button 하나만 표시한다.

우선순위는 다음과 같다.

1. 가장 오래된 미완료 마감
2. 작성 중인 마감 이어서 하기
3. 현재 영업일 마감 시작
4. 마감 결과 확인
5. 이번 달 경영성과 보기

동시에 여러 개의 강한 버튼을 표시하지 않는다.

---

# 13. 모바일 하단 메뉴

다음 다섯 개를 고정한다.

1. 홈
2. 마감
3. 비용
4. 경영성과
5. 더보기

기존의 `매출입력` 메뉴는 `마감` 안으로 통합하는 것을 기본 방향으로 한다.

매출만 직접 수정해야 하는 경우,
마감 상세화면에서 접근할 수 있다.

---

# 14. 사용자에게 보여주는 주요 용어

사용자 UI에서는 다음 용어를 사용한다.

- 마감
- 마감 시작
- 마감 완료
- 마감 결과
- 영업일
- 예상 플랫폼 공제액
- 예상 정산금액
- 이번 달 예상 이익

내부 개발 용어는 다음과 같이 유지할 수 있다.

- closing
- closing_status
- settlement
- business_day

# 15. 데이터 요청 항목

Home API는 최소한 다음 데이터를 반환해야 한다.

## 사업장

- business_name
- timezone
- closing_time

## 현재 영업일

- target_business_day
- closing_status
- current_step
- incomplete_business_days

## 마지막 완료 마감

- business_day
- total_sales
- expected_platform_deduction
- expected_settlement
- delivery_cost
- expense_total

## 이번 달

- total_sales
- expected_platform_deduction
- expected_settlement
- expense_total
- expected_operating_profit
- operating_profit_rate

## 차트

- recent_sales
- channel_sales_ratio

# 16. 성공 기준

Home 화면은 다음 기준을 만족해야 한다.

- 사용자가 3초 안에 현재 마감 상태를 이해한다.
- 사용자가 5초 안에 다음 행동을 찾는다.
- 마지막 완료 영업일 성적을 즉시 확인할 수 있다.
- 이번 달 총매출과 예상 이익을 10초 안에 이해한다.
- 미완료 마감이 있으면 날짜를 직접 찾지 않고 바로 시작할 수 있다.
- 신규 사용자도 설명 없이 첫 마감을 시작할 수 있다.
- 한 화면에서 강한 Primary Button은 하나만 보인다.
