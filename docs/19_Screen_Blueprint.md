# AI Business Agent Web Screen Blueprint

## 1. 문서 목적

이 문서는 실제 화면을 구현하기 전에 각 화면의 목적, 핵심 정보, 사용자의 행동을 정의한다.

모든 화면은 모바일을 기준으로 설계한다.

핵심 원칙은 다음과 같다.

```text
질문
↓
입력
↓
결과
```

사용자는 생각하지 않고 다음 행동을 알 수 있어야 한다.

---

# 2. Home

## 목적

- 현재 영업 상태를 5초 안에 이해한다.
- 오늘 해야 할 일을 바로 시작한다.

## Primary Action

오늘 마감 시작

## Secondary Action

경영성과 보기

## 핵심 정보

- 오늘 해야 할 일
- 오늘 마감 진행률
- 오늘 예상 정산금
- 이번 달 요약

---

# 3. 마감

## 목적

하루 마감을 가장 빠르게 완료한다.

## Primary Action

매출 확인

## 화면 정보

- 매출 상태
- 비용 상태
- 배달대행사 상태
- 진행률
- 예상 정산금

---

# 4. 플랫폼별 매출 입력

## 목적

생각하지 않고 플랫폼별 매출을 입력한다.

## Primary Action

저장

## 입력 항목

플랫폼에 따라

- 선결제
- 카드
- 현금
- 배달 매출
- 주문 수

---

# 5. 매출 정산 확인

## 목적

입력한 매출을 확인하고 예상 정산 결과를 확인한다.

## Primary Action

매출 확인 완료

## 화면 표시

플랫폼별

- 총매출
- 예상 공제액
- 예상 정산금액

전체

- 총매출
- 예상 공제액
- 예상 정산금액

---

# 6. 비용 확인

## 목적

오늘 발생한 비용을 확인한다.

## Primary Action

비용 추가

## Secondary Action

다음 단계

---

# 7. 배달대행사 확인

## 목적

예수금과 사용 금액을 확인한다.

## Primary Action

확인 완료

---

# 8. 마감 완료

## 목적

오늘 마감을 끝냈다는 확신을 준다.

## Primary Action

Home

## Secondary Action

경영성과

---

# 9. 경영성과

## 목적

이번 달 성과를 이해한다.

## Primary Action

월 변경

## 화면 정보

- 총매출
- 예상 공제액
- 예상 정산금액
- 영업비용
- 영업이익
- 플랫폼 비중

---

# 10. 더보기

## 목적

서비스 설정으로 진입한다.

## Primary Action

내 가게 설정

---

# 11. 내 가게 설정

## 목적

사업장 운영 기준을 관리한다.

## Primary Action

매출채널 설정

---

# 12. 매출채널 설정

## 목적

플랫폼 계약 수수료를 관리한다.

## Primary Action

플랫폼 선택

플랫폼

- 배달의민족
- 쿠팡이츠
- 요기요
- 땡겨요
- 일반결제

---

# 13. 플랫폼 설정

## 목적

실제 계약 수수료를 입력한다.

## Primary Action

저장하기

## Secondary Action

기본값으로 되돌리기

## 표시 항목

- 중개수수료율
- 결제수수료율
- 카드수수료율
- 건당 배달료

플랫폼에 따라 필요한 항목만 표시한다.

저장 후 정산 화면의 예상 공제액과 예상 정산금액이 즉시 변경된다.

---

# 14. 모든 화면 공통 원칙

모든 화면은 다음 순서를 따른다.

```text
입력
↓
저장
↓
계산
↓
결과
```

사용자가 가장 먼저 봐야 하는 숫자는

1. 예상 정산금액
2. 총매출
3. 예상 공제액

이다.

내부 계산 방식보다 사용자의 다음 행동이 먼저 보여야 한다.
