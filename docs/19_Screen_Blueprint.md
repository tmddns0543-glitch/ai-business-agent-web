# AI Business Agent Web Screen Blueprint

## 1. 문서 목적

이 문서는

실제 화면을 만들기 전에

모든 화면의 목적과

사용자의 행동을 정의한다.

이 문서를 기준으로

Figma를 제작한다.

---

# 2. Home

목적

사용자가

현재 영업일 상태를

5초 안에 이해한다.

사용자가

다음 행동을

바로 시작한다.

Primary Action

오늘 마감 시작

Secondary Action

경영성과 보기

---

# 3. 마감 시작

목적

사용자가

현재 영업일을

확인한다.

Primary Action

마감 시작

---

# 4. 매출 입력

목적

사용자가

생각하지 않고

매출을 입력한다.

Primary Action

다음

---

# 5. 매출 요약

목적

사용자가

입력 내용을

확인한다.

Primary Action

저장

Secondary

수정

---

# 6. 비용 확인

목적

오늘 발생한 비용을

확인한다.

Primary

비용 추가

Secondary

다음

---

# 7. 배달대행사 확인

목적

예수금이 맞는지 확인한다.

Primary

확인 완료

---

# 8. 마감 완료

목적

오늘 하루를 끝냈다는

성취감을 준다.

Primary

Home

Secondary

경영성과

---

# 9. 경영성과

목적

이번달을 이해한다.

Primary

월 변경

Secondary

상세보기

---

# 10. 설정

목적

환경을 관리한다.

Primary

저장

---

# 11. 피드백

목적

불편한 점을

바로 보낸다.

Primary

전송

---

# 12. 모든 화면 공통

모든 화면은

질문

↓

입력

↓

결과

순서를 따른다.

사용자가

생각하면 실패다.

프로그램이

다음 행동을 안내해야 한다.
