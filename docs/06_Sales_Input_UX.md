# AI Business Agent Web Sales Input UX

## 1. 문서 목적

이 문서는 웹서비스의 매출입력 구조, 입력 단계, 저장 방식, 수정 방식과 결과 화면을 정의한다.

매출입력은 다음 두 가지 방식을 제공한다.

1. 가이드 입력
2. 빠른 입력

가이드 입력은 처음 사용하는 사용자와 입력 구조가 익숙하지 않은 사용자를 위한 방식이다.

빠른 입력은 반복 사용에 익숙한 사용자가 한 화면에서 빠르게 입력하기 위한 방식이다.

---

# 2. 핵심 UX 목표

매출입력은 다음 조건을 만족해야 한다.

- 처음 사용하는 사용자도 별도 설명 없이 입력할 수 있어야 한다.
- 모바일에서 한 손으로 입력할 수 있어야 한다.
- 사용하지 않는 채널은 표시하지 않는다.
- 0원과 미입력을 명확히 구분한다.
- 중간에 종료해도 입력 내용이 사라지지 않는다.
- 저장 전에 전체 내용을 확인할 수 있다.
- 입력 완료 직후 예상 수수료와 예상 정산금액을 보여준다.
- 기존 입력을 쉽게 수정할 수 있다.
- 반복 사용자는 3분 안에 입력을 완료할 수 있어야 한다.

---

# 3. 입력 방식 선택

매출입력 화면 진입 시 다음 두 방식을 보여준다.

## 가이드 입력

설명:

채널별로 하나씩 확인하며 입력합니다.

추천 대상:

- 처음 사용하는 사용자
- 매출채널이 많은 사용자
- 입력 누락을 줄이고 싶은 사용자

버튼:

가이드 입력 시작

## 빠른 입력

설명:

사용 중인 모든 매출채널을 한 화면에서 입력합니다.

추천 대상:

- 입력 구조에 익숙한 사용자
- 반복 입력 시간을 줄이고 싶은 사용자

버튼:

빠른 입력 시작

사용자가 마지막에 선택한 방식을 저장하고 다음 진입 시 기본값으로 보여준다.

---

# 4. 날짜 선택

매출입력의 첫 단계에서 정산 날짜를 확정한다.

기본값:

오늘 날짜

사용자는 과거 날짜를 선택할 수 있다.

미래 날짜는 선택할 수 없다.

화면 표시:

정산할 날짜를 확인해주세요.

2026년 7월 12일

버튼:

다음

같은 날짜의 데이터가 이미 존재하면 다음 안내를 표시한다.

이미 입력된 매출이 있습니다.

버튼:

- 기존 내역 보기
- 수정하기
- 다른 날짜 선택

---

# 5. 가이드 입력 방식

## 5.1 기본 구조

가이드 입력은 채널별 묶음 방식으로 진행한다.

한 화면에는 하나의 플랫폼 또는 하나의 일반매출 묶음만 표시한다.

예시:

1단계 배달의민족
2단계 요기요
3단계 쿠팡이츠
4단계 땡겨요
5단계 일반 매출
6단계 전체 요약
7단계 저장 완료

사용자가 설정에서 활성화하지 않은 플랫폼은 건너뛴다.

---

# 6. 진행 상태 표시

화면 상단에 다음 정보를 표시한다.

- 현재 단계
- 전체 단계 수
- 현재 채널명
- 진행률

예시:

2 / 5

요기요

40% 완료

진행률은 시각적인 진행 바로 표시한다.

---

# 7. 배달의민족 입력 구조

배달의민족을 사용하는 사용자에게 다음 항목을 표시한다.

## 기본 항목

- 배달의민족 선결제 매출
- 배달의민족 카드 매출
- 배달의민족 현금 매출
- 배민원 매출
- 배민원 주문 수

## 입력 예시

제목:

배달의민족 매출을 입력해주세요.

입력 항목:

선결제 매출  
[420,000원]

카드 매출  
[85,000원]

현금 매출  
[30,000원]

배민원 매출  
[210,000원]

배민원 주문 수  
[14건]

버튼:

- 모두 0원
- 이전
- 다음

## 설명 문구

선결제:

배달의민족 앱에서 결제된 주문의 매출을 입력해주세요.

카드:

매장에서 카드로 결제된 배달의민족 주문 매출을 입력해주세요.

현금:

현금으로 결제된 배달의민족 주문 매출을 입력해주세요.

배민원:

배민원 주문의 총매출을 입력해주세요.

주문 수:

주문당 평균매출과 수수료 분석에 사용됩니다.

---

# 8. 요기요 입력 구조

다음 항목을 표시한다.

- 요기요 선결제 매출
- 요기요 카드 매출
- 요기요 현금 매출
- 요기배달 매출
- 요기배달 주문 수

화면 구성과 버튼은 배달의민족 입력과 동일하게 유지한다.

---

# 9. 쿠팡이츠 입력 구조

다음 항목을 표시한다.

- 쿠팡이츠 매출
- 쿠팡이츠 주문 수

쿠팡이츠 입력 항목이 적으므로 한 화면에서 입력한다.

---

# 10. 땡겨요 입력 구조

다음 항목을 표시한다.

- 땡겨요 선결제 매출
- 땡겨요 카드 매출
- 땡겨요 현금 매출

필요 시 향후 배달유형과 주문 수 항목을 추가할 수 있다.

---

# 11. 일반 매출 입력 구조

다음 항목을 표시한다.

- 일반 카드 매출
- 현금 매출
- 계좌이체 매출
- 포장 주문 매출

포장 주문 매출은 설정에서 사용하는 경우에만 표시한다.

---

# 12. 0원과 미입력의 구분

0원은 매출이 발생하지 않았다는 의미다.

미입력은 아직 확인하지 않았다는 의미다.

따라서 빈 입력칸을 자동으로 0원 처리하지 않는다.

입력칸 상태:

- 빈값: 확인되지 않음
- 0원: 매출 없음 확인
- 양수: 매출 발생

다음 버튼을 누를 때 빈 항목이 있으면 안내한다.

확인하지 않은 항목이 있습니다.

버튼:

- 빈 항목 확인하기
- 빈 항목을 모두 0원으로 처리
- 취소

사용자가 실수로 누락하는 것을 막기 위해 명확한 선택을 받는다.

---

# 13. 금액 입력 규칙

- 입력값은 0 이상의 정수만 허용한다.
- 음수는 허용하지 않는다.
- 숫자 입력 즉시 천 단위 쉼표를 표시한다.
- 모바일에서는 숫자 키패드를 사용한다.
- 원 단위로 입력한다.
- 복사한 숫자에 쉼표가 포함되어 있어도 정상 처리한다.
- 잘못된 문자는 자동 제거한다.

예시:

입력값  
420000

표시값  
420,000원

---

# 14. 주문 수 입력 규칙

- 0 이상의 정수만 허용한다.
- 소수는 허용하지 않는다.
- 주문 수가 0인데 매출이 있는 경우 확인 안내를 표시한다.
- 주문 수가 있는데 매출이 0원인 경우 확인 안내를 표시한다.

안내 예시:

매출은 입력되어 있지만 주문 수가 0건입니다.

입력값을 다시 확인해주세요.

---

# 15. 자동 임시저장

사용자가 다음 버튼을 누를 때마다 현재 단계의 입력값을 임시저장한다.

임시저장 상태:

- 작성 중
- 최종 저장 전
- 완료

사용자가 브라우저를 닫거나 다른 페이지로 이동해도 작성 중 데이터를 유지한다.

다시 매출입력 화면에 들어오면 다음 안내를 표시한다.

작성 중인 매출입력이 있습니다.

2026년 7월 12일

마지막 입력 단계:
쿠팡이츠

버튼:

- 이어서 입력
- 처음부터 다시 입력
- 작성 중 내용 삭제

---

# 16. 이전 단계 이동

사용자는 이전 버튼을 눌러 앞 단계로 돌아갈 수 있다.

이전 단계로 돌아가도 입력값은 유지한다.

수정 후 다음 버튼을 누르면 변경된 내용으로 다시 임시저장한다.

---

# 17. 중간 종료

사용자가 입력 도중 홈이나 다른 메뉴로 이동하면 안내를 표시한다.

지금까지 입력한 내용은 임시저장됩니다.

나중에 이어서 입력할 수 있습니다.

버튼:

- 나가기
- 계속 입력하기

---

배달의민족

매출  
745,000원

62,400원

예상 정산금액  
682,600원

수수료 상세는 접이식 영역으로 제공한다.

상세 항목 예시:

- 중개수수료

- 카드수수료

# 21. 빠른 입력 방식

빠른 입력은 활성화된 모든 채널을 한 화면에 표시한다.

## 화면 구조

- 날짜 선택
- 채널별 접이식 카드
- 전체 합계
- 저장 버튼

## 채널별 카드

배달의민족  
[펼치기 / 접기]

카드 안:

- 선결제 매출
- 카드 매출
- 현금 매출
- 배민원 매출
- 배민원 주문 수

사용자가 입력한 채널은 자동으로 펼쳐진 상태를 유지한다.

## 주요 버튼

- 모두 0원
- 임시저장
- 입력 요약
- 저장 완료
- 가이드 입력으로 전환

---

# 22. 기존 데이터 수정

기존 날짜의 매출을 수정할 때는 저장된 값을 먼저 불러온다.

화면 상단에 다음 상태를 표시한다.

2026년 7월 12일 매출 수정 중

사용자는 가이드 입력 또는 빠른 입력 중 하나를 선택할 수 있다.

수정 저장 전 변경 내용을 보여준다.

예시:

변경 전  
배민 선결제 420,000원

변경 후  
배민 선결제 450,000원

버튼:

- 수정 저장
- 변경 취소

---

# 23. 삭제

전체 날짜의 매출 삭제는 쉽게 노출하지 않는다.

수정 화면 하단의 위험 영역에 배치한다.

안내:

이 날짜의 매출입력을 삭제하면 경영성과 계산에서도 제외됩니다.

확인 방식:

- 삭제 동의 체크
- 삭제 입력
- 최종 삭제 버튼

삭제된 데이터는 즉시 완전 삭제하지 않고 일정 기간 복구 가능한 상태로 보관한다.

---

# 24. 저장 실패 처리

저장에 실패해도 입력값을 유지한다.

안내:

매출을 저장하지 못했습니다.

입력한 내용은 그대로 유지되고 있습니다.

버튼:

- 다시 저장
- 임시저장 후 나가기

---

# 25. 네트워크 오류 처리

네트워크가 불안정하면 현재 기기에 임시저장한다.

안내:

인터넷 연결이 불안정합니다.

입력한 내용은 기기에 임시저장되었습니다.

연결이 복구되면 다시 저장해주세요.

---

# 26. 성공 기준

가이드 입력 UX는 다음 기준을 만족해야 한다.

- 첫 사용자가 별도 설명 없이 입력 완료
- 활성 채널 4개 기준 5분 이내 완료
- 두 번째 사용부터 3분 이내 완료
- 빈 항목 누락률 5% 이하
- 입력 중 이탈 후 복귀 가능
- 저장 실패 시 데이터 손실 없음
- 입력 완료 직후 예상 정산금액 확인 가능
