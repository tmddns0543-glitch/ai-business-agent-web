# AI Business Agent Web Delivery Agency UX

## 1. 문서 목적

이 문서는 배달대행사 예수금, 배달비 사용, 캐시 충전,
현금주문 캐시입금, 환급 및 잔액 조정의 입력 방식과
배달대행사 현황 화면을 정의한다.

배달대행사 현황은 단순 비용 화면이 아니라,
배달대행사 캐시 또는 예수금의 입출금 흐름을 관리하는 화면이다.

---

# 2. 핵심 원칙

배달대행사 예수금은 다음 공식으로 계산한다.

현재 예수금
= 기초 예수금
+ 캐시 충전
+ 현금주문 캐시입금
+ 환급
± 잔액 조정
- 배달비 사용

사용자는 모든 금액을 양수로 입력한다.

프로그램은 거래유형에 따라
예수금 증가 또는 감소 방향을 자동 적용한다.

---

# 3. 배달대행사 거래유형

## 3.1 캐시 충전

매장이 배달대행사에 금액을 보내
캐시 또는 예수금을 충전한 거래다.

처리 결과:

- 예수금 증가
- 비용 영향 없음
- 현금 또는 계좌자산 감소

초기 웹 MVP에서는 현금흐름 분석과 직접 연결하지 않고,
배달대행사 예수금 증가만 반영한다.

---

## 3.2 배달비 사용

배달이 수행되면서 배달대행사 캐시가 차감된 거래다.

처리 결과:

- 예수금 감소
- 배달대행 비용 증가

예시:

배달비 사용 30,200원

예수금 영향:

-30,200원

비용 영향:

+30,200원

---

## 3.3 현금주문 캐시입금

플랫폼 주문이 현금결제로 발생했으나,
배달기사가 매장에 현금을 전달하지 못하고
해당 물건값을 배달대행사 캐시로 지급한 거래다.

처리 결과:

- 예수금 증가
- 비용 영향 없음
- 매출 추가 반영 없음

현금 주문 매출은 매출입력에서 이미 반영되므로,
캐시입금 시 추가 매출로 다시 계산하지 않는다.

예시:

현금주문 캐시입금 50,000원

예수금 영향:

+50,000원

비용 영향:

0원

매출 영향:

0원

---

## 3.4 환급 또는 취소

잘못 차감된 배달비나 취소된 배달비가
배달대행사 캐시로 반환된 거래다.

처리 결과:

- 예수금 증가
- 배달대행 비용 감소

예시:

배달비 환급 4,000원

예수금 영향:

+4,000원

비용 영향:

-4,000원

---

## 3.5 잔액 조정

실제 배달대행사 잔액과
프로그램 계산 잔액의 차이를 맞추는 거래다.

사용자는 다음 중 하나를 선택한다.

- 잔액 증가
- 잔액 감소

처리 결과:

- 예수금 증가 또는 감소
- 비용 영향 없음이 기본값
- 필요 시 조정 사유를 필수 입력

잔액 조정은 일반 입력보다 신중하게 사용하도록 한다.

---

# 4. 배달대행사 현황 화면

## 4.1 화면 목적

사용자가 다음 정보를 바로 확인할 수 있도록 한다.

1. 현재 배달대행사 예수금
2. 오늘 배달비 사용액
3. 오늘 캐시 유입액
4. 최근 거래내역
5. 잔액 변화

---

# 5. 화면 상단 요약

배달대행사별로 카드 형태로 표시한다.

예시:

바로고

현재 예수금
274,000원

오늘 배달비 사용
30,200원

오늘 캐시 유입
50,000원

오늘 순변동
+19,800원

---

# 6. 오늘 순변동 계산

오늘 순변동은 다음과 같이 계산한다.

오늘 순변동
= 오늘 예수금 증가 합계
- 오늘 예수금 감소 합계

예시:

배달비 사용
-30,200원

현금주문 캐시입금
+50,000원

오늘 순변동
+19,800원

---

# 7. 거래 추가

화면 상단 또는 하단에 다음 버튼을 제공한다.

배달대행사 거래 추가

버튼을 누르면 거래유형을 선택한다.

선택 항목:

- 캐시 충전
- 배달비 사용
- 현금주문 캐시입금
- 환급 또는 취소
- 잔액 조정

---

# 8. 거래 입력 화면

## 8.1 공통 입력항목

- 배달대행사
- 거래유형
- 날짜
- 금액
- 메모

## 8.2 선택 입력항목

현금주문 캐시입금의 경우:

- 관련 매출 날짜
- 관련 주문 메모

환급 또는 취소의 경우:

- 원거래 선택
- 환급 사유

잔액 조정의 경우:

- 증가 또는 감소 방향
- 조정 사유

---

# 9. 거래유형별 화면 문구

## 캐시 충전

질문:

얼마를 충전했나요?

설명:

배달대행사에 실제로 충전한 금액을 입력해주세요.

## 배달비 사용

질문:

배달대행비로 얼마가 사용되었나요?

설명:

오늘 또는 선택한 날짜에 차감된 배달대행비를 입력해주세요.

## 현금주문 캐시입금

질문:

현금주문 대금으로 캐시가 얼마 입금되었나요?

설명:

배달기사가 매장에 현금을 전달하지 못하고
배달대행사 캐시로 지급한 금액을 입력해주세요.

## 환급 또는 취소

질문:

얼마가 반환되었나요?

설명:

잘못 차감되거나 취소된 배달비가 반환된 금액을 입력해주세요.

## 잔액 조정

질문:

예수금 잔액을 어떻게 조정할까요?

선택:

- 잔액 증가
- 잔액 감소

---

# 10. 최근 거래내역

최근 거래는 최신순으로 표시한다.

예시:

2026-07-12

현금주문 캐시입금
+50,000원

배달비 사용
-30,200원

2026-07-11

캐시 충전
+200,000원

---

# 11. 거래 표시 규칙

예수금 증가 거래는 +로 표시한다.

예시:

+50,000원

예수금 감소 거래는 -로 표시한다.

예시:

-30,200원

사용자가 직접 부호를 입력하지 않고,
프로그램이 거래유형에 따라 부호를 표시한다.

---

# 12. 거래 상세

거래를 선택하면 상세정보를 보여준다.

표시 내용:

- 배달대행사
- 거래유형
- 거래일
- 금액
- 예수금 영향
- 비용 영향
- 관련 매출일
- 메모
- 생성일
- 수정일

버튼:

- 수정
- 삭제

---

# 13. 거래 수정

수정할 수 있는 항목:

- 날짜
- 거래유형
- 금액
- 메모
- 관련 매출일
- 잔액 조정 방향

수정 저장 후 예수금과 비용을 다시 계산한다.

---

# 14. 거래 삭제

거래를 삭제하면 해당 거래의 예수금 및 비용 영향도 제거한다.

안내:

이 거래를 삭제하면 배달대행사 예수금과
경영성과 계산이 변경됩니다.

삭제는 확인 절차를 거친다.

---

# 15. 비용입력과의 연결

배달비 사용 거래는 비용에도 자동 반영한다.

배달대행사 현황에서 배달비 사용을 입력하면:

- 배달대행사 예수금 감소
- 비용 거래 자동 생성

비용입력 화면에서 배달대행비를 입력할 경우에도
배달대행사 거래와 연결할 수 있어야 한다.

같은 거래가 비용과 배달대행사 화면에
중복 저장되지 않도록 하나의 원본 거래를 공유한다.

---

# 16. 현금주문 캐시입금과 매출 연결

현금주문 캐시입금은 매출을 새로 생성하지 않는다.

현금 매출은 이미 매출입력 화면에서 기록된다.

현금주문 캐시입금은
해당 매출대금을 실제 현금이 아닌
배달대행사 캐시로 받은 사실만 기록한다.

따라서:

- 매출 중복 반영 없음
- 비용 영향 없음
- 예수금만 증가

---

# 17. 기초 예수금 설정

배달대행사를 처음 등록할 때
현재 보유한 예수금 잔액을 입력할 수 있다.

입력항목:

- 기준일
- 기초 예수금
- 메모

이 값은 이후 거래 계산의 시작점이 된다.

---

# 18. 대행사 여러 곳 사용

한 사업장에서 여러 배달대행사를 사용할 수 있다.

예시:

- 바로고
- 생각대로
- 만나플러스
- 부릉
- 지역 배달대행사

각 대행사별로 예수금과 거래내역을 별도 관리한다.

---

# 19. 홈 화면 반영

홈에는 다음 정보를 요약 표시할 수 있다.

배달대행사 예수금

바로고
274,000원

생각대로
85,000원

예수금이 설정한 기준보다 낮으면 안내한다.

예시:

바로고 예수금이 부족할 수 있습니다.

---

# 20. 경영성과 반영

경영성과에는 배달비 사용과 환급만 반영한다.

반영 대상:

- 배달비 사용
- 배달비 환급

반영하지 않는 대상:

- 캐시 충전
- 현금주문 캐시입금
- 비용 영향 없는 잔액 조정

---

# 21. 데이터 검증

다음 조건을 검증한다.

- 금액은 0보다 큰 정수
- 거래유형 필수
- 배달대행사 필수
- 날짜 필수
- 잔액 조정은 방향 필수
- 현금주문 캐시입금은 비용에 반영하지 않음
- 배달비 사용은 비용에 자동 반영
- 같은 원본 거래를 중복 저장하지 않음

---

# 22. 성공 기준

배달대행사 현황 UX는 다음 기준을 만족해야 한다.

- 현재 예수금을 즉시 이해할 수 있음
- 거래유형별 증가·감소가 명확함
- 사용자가 음수를 직접 입력하지 않음
- 현금주문 캐시입금을 정확히 처리
- 배달대행비가 경영성과에 중복 반영되지 않음
- 여러 배달대행사를 각각 관리 가능
