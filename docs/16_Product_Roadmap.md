# AI Business Agent Product Roadmap

## 1. 문서 목적

이 문서는 AI Business Agent가 어떤 순서로 성장할 것인지 정의한다.

AI Business Agent는 모든 기능을 한 번에 제공하는 프로그램이 아니다.

외식업 점주가 매일 사용하는 핵심 기능부터 시작하여 단계적으로 전문가(Agent)를 추가하는 방식으로 발전한다.

---

# 2. 제품 철학

AI보다 먼저 정확한 데이터를 만든다.

모든 Agent는 실제 입력된 사업 데이터를 기반으로만 판단한다.

추측보다 실제 데이터,
감정보다 숫자,
일회성보다 반복 가능한 운영을 우선한다.

# 4. Phase 1 (현재 MVP)

## 목표

외식업 점주의 하루 마감을 5분 안에 끝낼 수 있도록 한다.

## 현재 구현 완료

### 마감

- Home
- 마감 진행률
- 매출 확인
- 비용 확인(기반 구조)
- 배달대행사 확인(기반 구조)

### 매출

- 플랫폼별 매출 입력
- 플랫폼별 LocalStorage 저장
- 전체 총매출 계산
- 플랫폼별 주문 수 관리

### 정산

- 플랫폼별 계약 수수료 설정
- business-fee-settings 저장
- 공통 정산 계산 엔진
- 채널별 계산
- 플랫폼별 계산
- 전체 계산
- 예상 공제액
- 예상 정산금액

### 설정

- 더보기
- 내 가게 설정
- 매출채널 설정
- 배달의민족 설정
- 쿠팡이츠 설정
- 요기요 설정
- 땡겨요 설정
- 일반결제 설정
- 플랫폼별 기본값 복원

### 사용자 경험

- 설정 변경 즉시 정산 반영
- 모바일 중심 UI
- 공통 계산 로직
- 플랫폼별 계약 조건 반영

## Phase 1 남은 작업

- 비용 입력 완성
- 배달대행사 거래
- 월별 경영성과
- 실제 입금액 비교
- 영업일 관리
- SQLite/DB 전환
- 사용자 계정

## 성공 기준

사용자가

- 매출 입력
- 플랫폼 수수료 설정
- 예상 정산 확인
- 비용 확인

까지 하루 5분 안에 완료할 수 있어야 한다.

CEO가 만든 데이터를 분석하여 경영 의사결정을 지원한다.

- 광고비율
- 플랫폼 수수료 분석
- 영업이익률
- 전월 비교

- 개선 제안
- 정산 오차 분석

# 6. Phase 3 (CMO)

## 목표

### 주요 기능

- 광고 효율 분석
- 플랫폼별 효율
- 리뷰 분석

- 마케팅 제안
- 제휴 추천

# 7. Phase 4 (COO)

## 목표

운영 효율을 높인다.

### 주요 기능

- 재고 분석
- 발주 추천
- 생산 계획
- 폐기율 분석
- 원재료 예측
- 운영 효율 분석

---

# 8. Phase 5 (CSO)

## 목표

사업의 장기 전략을 제안한다.

### 주요 기능

- 점포 확장
- 투자 시점
- 신규 메뉴
- 인력 운영
- 장기 목표 관리

---

# 9. Agent 관계

```text
CEO
↓

CFO
↓

CMO
↓

COO
↓

CSO
```

모든 Agent는 CEO가 만든 데이터를 기반으로 동작한다.

---

# 10. 개발 우선순위

```text
① CEO 안정화
↓

② 베타 사용자 확보
↓

③ CFO 개발
↓

④ CMO 개발
↓

⑤ COO 개발
↓

⑥ CSO 개발
```

현재 프로젝트는 CEO Phase의 핵심 기능을 완성하고 베타 테스트를 준비하는 단계이다.

---

# 11. 기능 추가 기준

새 기능은 다음 조건을 만족해야 한다.

1. 실제 데이터 기반인가
2. 점주의 의사결정을 돕는가
3. 반복 사용되는가
4. UX를 복잡하게 만들지 않는가
5. 베타 사용자에게 검증 가능한가

---

# 12. 장기 목표

AI Business Agent는 단순 회계 프로그램이 아니다.

최종 목표는 외식업 점주의 Business Operating System(Business OS)이 되는 것이다.

입력 → 저장 → 계산 → 분석 → 의사결정까지 하나의 흐름으로 연결되는 운영 플랫폼을 구축한다.
